## 2024-05-15 - Prevent Arbitrary Command Execution in IPC openPath Handlers
**Vulnerability:** The Electron `open-path` IPC handler previously executed external applications using an unverified input parameter `app` from the renderer via `execFile` without validating the executable name or path.
**Learning:** In desktop environments where developers need to open files in varying editors (e.g. VSCode, Cursor, Zed), a strict allowlist can break functionality. However, allowing absolute paths requires extreme care. Attackers could theoretically bypass simple directory constraints by pointing directly to built-in system shells (`bash`, `cmd`, `node`, `powershell`) to execute arbitrary malicious scripts or commands if the arguments string contained crafted inputs or file paths that were actually scripts.
**Prevention:** Rather than a strict allowlist, use a combination of a robust blocklist for inherently dangerous system interpreters/shells (`sh`, `bash`, `cmd`, `node`, `powershell`), path traversal prevention (`..`), and enforcing that the specified path resolves securely (`checkAppExists` and `resolveAppPath`) before executing it. Always validate input strings to secure `execFile`.
